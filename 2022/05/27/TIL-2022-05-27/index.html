수업 내용

SDLC(Software Development Life Cycle)

  • 소프트웨어를 계획하고, 개발하고, 시험 및 배포하는 과정 전체를 뜻한다.
  • 요구사항의 분석 → 설계 → 구현 → 테스트 → 운영 또는 유지보수

Model

  • Build & Fix : 다른 코스 없이 일단 만들고 고치고하는 방법
  • Prototype : 최소한의 요구사항 분석 후 임시 모델을 만들어 고객의 요구를 개발에 도입. 임시적으로 주 기능과 디자인을 어느정도 해놓고 클라이언트에게 보내 확인을 받는 법
  • Waterfall : 요구사항 분석 → 설계 → 구현 → 운용 이 순차적으로 진행되는 방식. 대규모 팀에 적합하며 각 단계를 완전히 완료하기 전까지 다음 단계로 넘어가지 않음.
  • Spiral : 어떠한 작업의 목표설정 → 위험분석 → 개발 및 검증 → 고객 평가 및 다음 단계 진행을 반복한다. 점진적으로 단계를 반복하여 위험을 최소화하며 대형 시스템의 구축에 유리하다.
  • Agile : 프로젝트의 생명주기동안 반복적인 개발을 촉진하는 모델이다. Waterfall의 과정을 짧게 짧게 반복하는 것인데 Waterfall이 전체 결과물이 대상이라면 Agile은 한 컴포넌트나 기능을 대상으로 하여 Waterfall 형식을 반복하는 것이다.

Scrum

  • 전체 기능 중 개발할 기능과 수정사항에 우선 순위를 부여한 뒤, 이 순서대로 Task 진행
  • 매일 15분의 일일 회의 진행
  • 1~4주 간의 기획, 개발, 리뷰 진행
  • Daily Scrum : 어제 한 일과 오늘 할 일, 이슈 등 매일 공유
  • TDD(Test Driven Development) : 테스트코드 먼저 작성 후 본 코드 작성

Wireframe

Section 별로 구분을 하여 UI를 어떻게 구성할지 대략적으로 그린 구조도 이미지

Usecase

사용자가 어떠한 경로나 링크를 통해 홈페이지의 각 페이지에 접근할 수 있는지를 보여주는 간단한 그림 구조

Wildcard & Navigate

  • props.path로 *를 넘겨주는 경우 모든 url이나 파일에 대응하는 것을 뜻한다.

  • : 해당 URL로 페이지를 이동시켜주는 컴포넌트

  • props.replace가 false인 경우

    • /로 접속
    • Main을 render
    • 클릭
    • Navigate에 의해 /hi로 이동
    • 뒤로가기 클릭
    • /로 이동

  • props.replace가 true인 경우

    • /로 접속
    • Main을 render
    • 클릭
    • Navigate에 의해 /hi로 이동
    • 뒤로가기 클릭
    • /abc로 이동

  • 즉 props.replace는 browser history에 영향을 끼친다.

React in CSS

  • CSS In JS : Javascript 코드로 css처럼 스타일링을 하는 방식
  • 왜 CSS In JS를 써야하는가

    • Global Namespace

      • 모든 스타일이 Global에 선언되어 중복되지 않는 Class 이름을 적용해야한다.

      • A.css 에서 div 태그 자체에 정의를 하면 해당 css 파일을 import 하지 않아도 적용됨

    • Dependencies

      • CSS에서 에러를 감지하기 어려움
      • css class의 이름을 변경할 때 js에서 알 방법이 없음
      • 파일간 css 코드를 이동하면 놓칠 수 있음
      • class name을 부여 후 css 코드 작성을 까먹을 수 있음

    • Dead Code Elimination

      • 사용하지 않는 코드 관리가 어려움
      • css는 js/html 과 직접적으로 연결되어 있지 않아 상대적 어려움

    • Minification

      • css 명이 string으로 관리되어 짧게 만들기 어렵다
      • 해결하기 위해선 webpack css module을 사용해야한다.

    • Sharing Constans

      • css와 js간 variable 공유가 어렵다

      • js 변수를 css 변수로 활용 시 불편함

    • Non-Deeterministic Resolution

      • css import가 비동기적으로 이루어짐
      • import 순서에 맞지 않게 작동할 수 있는 가능성이 있음

    • Breaking Isolation

      • css Selector 작성 시 안쪽의 component style까지 영향을 끼침

Styled

  • 백틱을 이용한 활용

  • 이렇게 만든 styled는 Component가 된다.

Props

  • ${} 문법을 이용한다.

  • styled 자체에서 문법 검사를 해주어서 undefined나 잘못된 값이 있다면 자동으로 제거해준다.

Animation
