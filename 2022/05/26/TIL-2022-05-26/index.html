    로그인 기능 만들기

    • 조건

      • id와 pw를 state, input으로 관리해야 합니다.
      • 로그인 버튼 클릭 시 id나 pw 둘 중 하나라도 값이 비어있다면, alert창을 통해 submit event를 종료시킵니다.
      • axios를 통신하는 동안 button을 다시 클릭할 수 없습니다.
      • axios 통신 결과에 따라 user state (redux)를 변경하거나,Error Msg를 지정합니다.
      • Error Msg는 있을 때에만 출력되며, 1.5초 뒤 사라집니다.
      • Error Msg가 사라지며 button을 다시 클릭할 수 있습니다.

    • API 명세서

      • 반드시 body(obj)를 같이 보내줘야 함.
        • body.id : 유저 id
        • body.password : 유저 pw
      • 로그인 실패 시
        • code : 400 = body 값이 비어 있을 때
        • code : 401 = 존재하지 않는 id일 때
        • code : 402 = 비밀번호가 틀렸을 때
      • 로그인 성공 시
        • code : 200
        • userInfo = object, user의 data가 넘어 옴

    • 코드

      • Login Component
      • View User (Mypage)
      • Store
      • userSlice

    회원가입 기능 만들기

    • 조건

      • 아이디 중복 체크 기능 제작
        (End Point 별도 전달)
      • ID 체크 응답(res.data)
        • 중복된 id가 없을 경우 : check = true
        • 중복된 id가 있을 경우 : check = false
      • id, name, password 3개의 정보를 담아서 보낸다. 회원가입을 진행하기 이전에 사용자가 입력한 id를 체크해서 중복된 id가 없을 경우에만 회원가입을 진행한다.
      • 회원가입 응답(res.data)
        • 회원가입 성공 : code = 200
        • 회원가입 실패 : AxiosError (중복된 ID가 있을 경우 에러발생)

    • 코드

      • Signup Component

    회고

    오랜만에 프로젝트 형식으로 진행된 실습을 하였습니다.

    실제로 로그인 기능과 그 결과를 보여주는 기능, 회원가입 기능과 아이디 중복체크 기능까지 해보며 원하는대로 작동하는 것을 보는 것은 언제나 즐겁습니다. 다만 아직까지 코드의 단축화에 대한 리팩토링은 항상 필요한 것 같습니다.

    굳이 data를 전송하지 않아도 되는 부분에 대해서도 피드백을 받고, 강의를 들으며 깨닫기도 하였습니다. 코드를 한 번 짜고 다시 한 번 되짚어보는 습관이 늘 필요한 것 같다 생각합니다.
