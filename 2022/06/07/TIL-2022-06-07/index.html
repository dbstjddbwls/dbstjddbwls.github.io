TypeScript Class

  • class ClassName{ }이란 문법을 통하여 생성한다.
  • 하나의 객체로 취급된다.
  • Class 내 요소 앞에 아무것도 적지 않으면 public이란 값이 자동으로 들어간다.
  • 해당 객체 내 요소 앞에 private라는 값을 붙히면 해당 Class 내에서만 접근이 가능하고 외부에서 접근은 불가능해지며 상속도 불가능해진다.
  • 해당 객체 내 요소 앞에 protected라는 값을 붙히면 해당 Class 내에만 접근이 가능하고 외부에서의 접근은 불가능해진다. 단 private와 다른 점은 상속은 가능해진다.
  • Class 내에서도 interface 구현이 가능하다. 단 사용하기 위해선 interface 내 명시된 값들이 모두 들어가야 오류가 나지 않는다.
  • 사용하기 위해선 ClassName 앞에 implements InterfaceName을 적어 사용한다.
  • class 앞이나 class 내 요소 앞에 abstract라는 값을 주는 것으로 특정 속성이나 메서드를 상속받는 class에서 필수적인 값으로 만들게할 수 있다.
    이를 추상 클래스라고 하며 이러한 클래스는 별도로 선언을 할 수 없고 다른 클래스의 하위 클래스로 들어가야한다.

Generic

  • Generic이란 함수에서 파라미터를 정의하듯이 타입을 파라미터처럼 정의할 수 있게된다.
  • 함수명 뒤에 처럼 작성한 뒤 어떠한 인자의 값에 GenericName을 적으면 쓸 수 있다.
  • 이후 해당 함수를 호출할 때 GenericName 부분에 타입을 넣어서도 사용 가능하다.
  • 인자 뒤에 [type]을 적으면 반환 값의 type으로 지정해준다.
  • Generic은 inerface의 값을 넣어 사용할 수도 있다. 이 경우 inerface 내 정의된 값 외엔 인자로 줄 수가 없게된다.
  • Generic은 inerface 자체에도 선언할 수 있다.
  • Generic Type의 경우 실제 적용시 삼항 연산자도 사용이 가능하다. 이러한 경우 return 값의 반환으로 어떤 것을 주냐에 따라 달라진다.

TypeScript의 Type

  • TypeScript의 Type은 Static Type으로 개발하는 중간 과정, 즉 코드 작성 때 Type을 확인한다.
  • Javascript의 Type은 Dynamic Type으로 개발할 때는 알 수 없고 런타임이 실행되야 확인할 수 있다.
  • TypeScript에선 JS에서 사용 가능한 타입에 추가적으로 열거 타입을 쓸 수 있게되었다.
  • TypeScript에서 사용 가능한 Type
    • 기본 Type
      • Boolean
      • Number
      • String
      • Null
      • Undefined
      • Symbol
      • Array
    • TS에서 추가된 열거형 Type
      • Any, Void, Never, Unknown
      • Enum
      • Tuple: Object 형

Primitive Type

  • Object나 레퍼런스의 형태가 아닌 실제의 값을 저장하는 자료형을 뜻한다.

  • ES2015 기준 6가지가 있는데 Boolean, Number, String, Symbol, Null, Undefined가 있다.

  • Typescript의 Primitive Type은 모두 소문자로 작성한다.

  • Symbol?

    • new Symbol로 사용 불가
    • Symbol을 함수로 사용하여 Symbol Type을 만들 수 있다.
    • Symbol을 쓰기 위해선 tsconfig.json 파일 내 lib 속성에 값으로 [ “ES2015”, “DOM” ] 값을 추가해줘야한다.
    • 해당 함수 내에 자료형 타입의 값을 담아 사용하며, 고유하고 수정 불가능한 값으로 만들어준다.
    • 주로 접근을 제어하는데 쓰는 경우가 많다

  • Tuple?

    • union은 둘 중 하나가 들어가는 합집합이라면 tuple은 동시에 선언할 수 있는 방법이다.
    • 선언한 타입 순서에 맞게 들어가야 오류가 나지 않으며 추가도 불가하다.

  • any?

    • 어떠한 타입도 들어갈 수 있는 경우 사용한다.
    • 단 타입에 따른 예기치 않은 오류가 날 수 있어 조심히 사용해야한다.
    • 그만큼 최대한 쓰지 않는게 핵심이며 컴파일 타임에서 타입 체크가 정상적으로 이뤄지지 않기 때문이다.

  • unknown?

    • any와 같이 무엇이 올 지 알수 없는 변수의 타입 선언에 쓴다.
    • any와의 차이점은 string이나 number 등에 바로 사용은 불가능하다.

  • never?

    • 함수를 선언할 때 사용하며, 아무것도 return 되지 않을 때 사용한다.
    • nerver 타입은 모든 타입에 할당할 수 있다.
    • 하지만 never 자체에는 어떠한 것도 할당이 불가하다. any조차 불가하다.
    • 주로 잘못된 타입을 넣는 실수를 막을 때 사용된다.

  • void?

    • return 되는 값이 없어야할 때 선언하는 type이다.
    • 같은 의미라하여 null을 넣어도 오류가 나나, return 값으로 undefined만을 넣을 수 있다.

코딩테스트란?

  • 기업이나 기관에서 인원 선발을 목적으로 시행되는 문제 풀이 시험
  • 문제 해결 역량 평가를 통하여 프로그래밍적 사고를 확인 가능하다.
  • 온라인 저지 (대회 또는 시험에 나올 문제를 시험해보는 곳) 사이트
    • 백준 온라인 저지
    • 프로그래머스
    • 코드업
    • 구름
  • 선생님 코딩테스트 깃허브 : https://github.com/ndb796/Python-Competitive-Programming-Team-Notes

코딩테스트 공부 순서

  • 1단계 : 프로그래밍 언어 문법 공부

  • 3단계 : 유형별 알고리즘 이론과 핵심 풀이
  • 4단계 : 백준 온라인 저지에서 문제 풀이

  • 6단계 : 프로그래머스 카카오 문제 풀기
  • 7단계 : 주요 알고리즘 문제 복습
