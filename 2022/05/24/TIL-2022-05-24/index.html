Next.js로 Blog 만들어보기

  • 참고 URL : https://nextjs.org/learn/basics/create-nextjs-app/setup

  • import Link Component

    • link를 넣는 컴포넌트로 a 태그와 같이 사용할 수 있다.

    • 두 태그의 차이점은 a 태그는 다른 페이지 항목을 처음부터 새로 불러들이는 것이라면, Link 태그는 기존 보고있는 페이지와 이동할 페이지를 비교하여 없는 파일이나 요소만 불러들여 화면에 출력한다. 즉 공통되는 파일이나 요소를 저장하는 이러한 점 때문에 페이지 이동 시의 깜빡임이 사라진다.
      이러한 것을 Code Splitting이라고 부른다.

    • 또 다른 기능으로는 prefetching이라는 기능이 있다.
      npm run build를 한 뒤 npm start를 하였을 때 보여지는 화면에 보여지는 정보만 가져오는 것이 아닌, 보여지는 화면에서 이동할 수 있는 링크의 페이지 정보까지 우선적으로 받아들인다. 이후 페이지 이동 시에는 chunk만 불러오게 된다. 이러한 기능 또한 a 태그만 사용해서는 불가능한 기능이다.

  • import Image Component

    • image 태그와 비슷한 기능을 하는 컴포넌트이다.

    • 속성을 동일하게 받으나 사용 사이즈에 맞춰 이미지 파일을 자동으로 최적화를 해주어 파일 크기도 줄어들게 된다.

  • MetaData

    • HTML에서 head 태그의 역할을 하는 기능이다.

    • 주로 란 태그를 써서 사용하며 안에 title 태그나 link favicon 태그 등을 사용하여 넣는다.

  • CSS Modules

    • React에서 CSS를 적용 시키는 방법 중 하나이다.

    • css 파일 자체로 적용 하는 것이 아닌, .module.css 파일로 만들어 적용하는 방법으로 특정 div나 컴포넌트의 className으로 module 파일 내 class 이름을 지정하면 자동으로 적용된다.

    • 해당 파일의 이름은 fileName.css가 아닌 fileName.module.css로 지정을 해야만 한다.

  • Global CSS

    • 모든 페이지에 전반적으로 들어갈 css 파일을 지정한다

    • 경로는 보통 처음 CNA 제작 시 자동으로 정해지며 pages 폴더 내 _app.js란 파일을 만들어 해당 파일 안에 Global CSS를 import 시킨다.

    • _app.js에 import된 css는 모든 파일에 공통적으로 자동 반영된다.

    • _app.js는 모든 컴포넌트를 한 번 감싸는 역할을 한다. 즉 css만이 아니라 js code도 공유가 된다.

    • SASS 또한 이용할 수 있다.

  • hydration
    페이지가 브라우저에 로딩되고 JS CODE가 실행되며 페이지가 인터렉팅하게 동작할 상태가 되는 과정이다.

  • pre-rendering

    • Next.js는 모든 페이지를 기본적으로 Pre Rendering을 하고 있다.
    • HTML 태그들을 먼저 우선적으로 pre-rendering을 한 뒤 이후 JS 파일들을 불러와서 적용시킨다. pre-rendering 이후 JS 파일을 적용시키는 과정을 위에서 말한 hydration이라고 부른다.
    • Next.js로 npm run start를 한 화면에서 js disabled를 하게되면 import나 alert 같은 코드들은 모두 제외되고 태그 형식의 컴포넌트나 이미지들만 보이게 된다.
    • 반면 create-react-app으로 만든 순수한 react-app에게 js disable은 치명적이다. React-app은 JS 실행으로 화면을 그리는 것이라 어떠한 화면도 나오지 않게 되기 때문이다.

  • getStaticProps()

    • 해당 함수를 통하여 마크다운이나 다른 폴더들을 읽어서 올 수 있다.

    • gray-matter라는 라이브러리로 파일의 해석을 진행해야 한다.

    • 따로 모듈화를 해두는 것이 좋으며 이러한 읽어본 파일들의 정보는 server 폴더 내에서 확인할 수 있다.

    • 이러한 기능은 build될 때 pre-rendering 된다.

    • 만약 사용자의 요청마다 외부의 데이터를 가져와야 한다면 getStaticProps가 아니라 getServerSideProps를 사용하면 된다.

  • Dynamic Routes

    • 파일을 읽고 불러들이고 만드는 역할을 하는 기능들이다.
    • getStaticPaths : [{ params : { id : string }}] 목록을 반환한다. params 안에는 dynamic routes key가 들어있어야한다.
    • getStaticProps: id 별로 md 파일에서 데티러를 읽어 page에 props로 전달하는 기능을 한다.
    • remark : npm으로 html에 md content를 보여주는 역할을 한다.
    • Catch-all Route : id를 하나만 받는 것이 아닌 여러 개를 받는 역할로 /pages/posts/[…id].js 이러한 방식으로 만들어두고 사용한다.
    • fallback : false면 404를 반환하고 true면 router.isFallback을 반환한다.
