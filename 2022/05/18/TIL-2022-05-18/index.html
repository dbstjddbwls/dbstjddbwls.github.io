수업 내용

Mocking 라이브러리

  • 모의 데이터(Mock)을 만들어서 활용하는 방식이다.
  • 통상적으로 data fetch를 해야하는 경우 통신을 통해 응답을 내려주는 서버가 있어야한다.
  • 서버가 없는 경우엔 API 요청으로 받을 데이터를 프론트에서 모킹하거나 서버의 역할의 해주는 무언가가 필요하다.
  • REST API나 GraphQL 또한 모킹이 가능하다.

Redux

  • 컴포넌트 사이의 전역 상태간 상태관리 시스템이다.
  • 사용법은 npx create-react-app my-app —template redux를 작성하여 사용
  • typescript를 사용하고 싶다면 redux부분에 redux-typescript를 덧붙힌다.
  • 이후 npm install react-redux를 실행시키거나 npm install @types/react-redux를 입력한다.
  • useSelect() : Provider로 감싼 store에서 state 데이터를 가져올 수 있다.
  • useDispatch() : useDispatch()란 함수 안에 넣으면 dispatch 함수를 만들 수 있다. 액션 객체나 속성값 자체를 전달하며 작동시킨다.
  • Provider : react에 store를 손쉽게 연동할 수 있게 해주는 컴포넌트이다. 해당 컴포넌트로 React 컴포넌트들을 감싸준다면 하위 컴포넌트들이 Provider을 통해 Redux의 store를 조정할 수 있다.
  • store : 스토어는 사태가 관리되는 공간이다. 컴포넌트와 스토어는 별개의 공간이 있어, 스토어 안에 앱에 필요한 정보나 상태를 저장한 뒤 이후 필요할 때 꺼내서 사용한다.

MobX

  • MobX는 redux와 다르게 코드의 양을 획기적이게 줄이기 위해 나왔다.
  • 심플한 형태로 이루어지기 위하여 보일러플레이트의 형식을 버렸다.
  • Action을 통하여 Derivative를 바꾸는 유일한 수단으로 사용된다.
