JS 데이터의 기능들

  • String.match() : 정규표현식을 인자로 받으며, 해당 표현식에 맞는 문자 앞까지 추출한다.
  • String.trim() : 해당 문자열 안에 포함된 공백 문자를 제거하여 반환한다.
  • Number.toFixed() : 해당 숫자값에 소수점이 있다면 인자로 받는 소숫점 자리에서 끊어서 문자형으로 반환한다.
  • parseInt() : 문자열을 받으면 숫자형만 추출하여 반환한다. 단 이 과정에서 소숫점은 사라지고 정수만 반환한다.
  • parseFloat() : parseInt와 같으나 소숫점 자리까지 보여준다.
  • Math() : 수학적인 상수와 함수를 위한 내장 객체이다.
  • Math.abs() : 주어진 숫자의 절대값을 반환
  • Math.ceil() : 주어진 숫자값을 올림처리를 함
  • Math.floor() : 주어진 숫자값을 내림처리를 함
  • Math.round() : 주어진 숫자값을 반올림처리를 함
  • Math.random() : 0~1 사이의 랜덤한 난수를 추출한다.
  • Array.find() : 주어진 판별 함수나 조건에 의해 매칭되는 첫 요소를 반환한다.

  • Array.forEach() : 주어진 배열의 item 갯수만큼 특정한 콜백 함수를 반복 실행한다.

  • Array.map() : 주어진 배열의 item 갯수만큼 특정한 콜백 함수를 반복 실행하여 해당 값을 배열로 반환한다.

  • Array.filter() : 주어진 배열의 item 갯수만큼 특정한 콜백 함수를 반복 실행하여 해당 함수 조건에 맞는 item을 배열로 새롭게 만들어 반환한다.

  • Array.findIndex() : 주어진 배열의 item 갯수만큼 특정한 콜백 함수를 반복 실행하여 해당 배열 내 조건에 맞는 item의 첫 번째 요소의 index number를 찾는다.

  • Array.includes() : 주어진 배열 내 인자로 받은 데이터가 존재하는지 확인한다.

  • Object.assign() : 인자로 받은 객체 데이터를 합쳐서 비교 또는 서술할 때 사용할 수 있다.

구조 분해 할당

어떠한 객체의 구조를 분해햐여 변수로 할당한다. 정의되어있지 않은 속성을 할당하면 undefined를 반환하게 된다. 또한 따로 값을 지정하여 추가할 수 있고, 만약 기존에 값이 있는 경우 변수에 할당한 값은 무효처리 된다.

Global Context

코드 구현을 위한 글로벌 환경을 설정하는 것. 정해진 내부 순서를 마친 뒤 다시 Global Context를 실행하여 다른 환경을 세팅한다.

Global Context 순서

  • 기본 window 관련 함수, 사용자 설정 함수 선언
  • var의 참조자 선언
  • Outer 환경 참조 및 this 객체 선언
  • Object 객체 선언, let이나 const에 변수에 값을 할당

let과 var의 차이점 (블록 스코프)

조건문 안에서 재선언 후 로컬이 아닌 전역에서 호출하였을 때 값이 달라진다.

let과 const의 경우 한 블록(동일한 중괄호) 안에 같은 이름으로 let 변수명을 또 한 번 선언하면 오류가 난다. 그러나 var의 경우엔 이러한 오류가 나지 않는다.

즉 전역 스코프냐 블록 스코프냐의 차이인데, let과 const는 블록이 다르면 새로운 화이트보드 위에 새로 그리게 되는 것이고, var는 그것이 조건문이나 반복문 밖에도 영향을 끼치게 되는 것이다.

단순히 var는 재선언, let이나 const의 경우 같은 블록 스코프에선 재선언이 불가하며 다른 블록일 경우 재선언보단 동일한 이름의 새로운 변수가 탄생하는 것이다.

이 경우 에러가 나오는데 이유는 초기화가 되지 않아서이다. var였으면 실행 전 지역 변수에 없으니 전역 변수에서 값을 가져왔을 것이지만, let이나 const의 경우 지역 내에서 전역 let이나 const의 값을 가져오고 싶다면 초기화를 우선적으로 해야한다.

다만 선언을 어떠한 형식으로 하냐에 따라 지역 변수가 없다면 전역 변수에서 받아오는 것이 가능하다.

요약) let과 const는 초기화가 되지 않는다. function name()으로 선언을 하면 null로 초기화가 된다.

함수 스코프

블록 스코프와 같이 function에서 받은 인자값은 외부에서 해당 인자값의 변수명으로 불러올 수 없다. 같은 인자의 이름을 사용하더라도 지역인자로만 사용되어 전역의 값으론 사용이 불가능하여 꼬이지가 않는 것이다.

호이스팅

위와 같이 const로 선언을 하였다면 const는 Global Context의 순서에 의해 const 선언이 함수 실행보다 뒤에 있어서 오류가 난다.

그러나 function의 선언의 경우 Global Context 순서 중 가장 첫 사이클, 준비 단계에 초기화 후 선언되기 때문에 (선언형 방식) 가능해진다.

호출이 선언보다 위에 있어도 Global Context의 순서에 따라 끌어오는 것을 호이스팅이라고 한다.

함수 코드 평가

함수의 단위, 즉 하나의 함수가 생길 때마다 함수 실행 컨텍스트가 생성된다. 가장 첫 단계인 콜스택에 하나씩 쌓이는 것이다.

만약 콜스택 1의 함수에서 a를 console.log를 하라 하였을 때 해당 함수 내에 지역 변수로 없다면 다른 함수나 전역으로 가서 찾게된다. 이후 함수가 실행되면 콜스택과의 관계를 끊어버리며 스택의 층이 변하게 된다. 이 때문에 함수 안에서 함수 밖에는 접근이 가능하나, 함수 밖에서 함수 안의 지역 변수에 접근을 할 수가 없다.

Javascript의 메모리

메모리는 크게 4가지 영역으로 분류되어있다.

  • 코드 영역 : 유저가 입력한 코드가 들어있는 영역
  • 스택 영역 : 입력한 코드에 따라 처리해야할 항목들을 순서대로 쌓아두는 영역
  • 데이터 영역 : 메모리의 주소가 담겨있고 데이터를 구조화시키는 영역
  • 힙(heap) 영역 : 데이터 영역에서 실행할 기능들이 담겨있는 영역
    버튼을 클릭시 객체를 생성하는 기능같은 기능들이 모여있다.

메모리 순서도

  • 스택 영역에 하나씩 콜을 해온다. = 콜스택
  • 코드를 실행할 환경을 셋팅한다.
  • 프로그램을 하는데 필요한 메모리 객체를 생성할 환경을 셋팅한다. (var, const, let, this)
  • window에 관련된 속성이나 기능을 셋팅한다. (console, alert, var b = {age:’20’} …)
  • function에 관련된 속성이나 기능을 셋팅한다. (prototype …)

요소 쌓임 순서 (Stack order)

어떤 요소가 사용자와 더 가깝게 있는지(위에 쌓이는지) 결정

  • 요소에 position 속성의 값이 있는 경우 위에 쌓임
  • 1번 조건이 같은 경우 (요소 중 position 속성이 다양하게 있는 경우) z-index 속성의 숫자 값이 높을 수록 위에 쌓임
  • z-index조차 같으면 html 구조 상 아래에 있는 요소가 더 위에 쌓임

FLEX

  • 특정한 요소에 display : flex를 추가하면 하위 요소가 수평정렬이 된다.
  • 부모 요소에 display : flex를 추가하면 해당 부모 요소는 flex container로 지정이 된다.
  • 자식 요소의 경우엔 flex items라고 지정이 되며, flex를 추가할 시 2가지 개념이 각 요소에 지정이 된다고 보면 된다.

Flex Container의 특성

  • display : flex와 inline-flex를 부모요소에 부여할 수 있다. flex는 flex-container들이 block 형식으로 쌓이고, inline-flex는 인라인 요소처럼 inline 형식으로 쌓인다.

  • flex-direction : row, row-reverse, column, column-reverse 란 속성을 사용하여 flex-item들이 어떤 정렬 형식으로 될지 결정한다.

  • flex-wrap : nowrap, wrap란 속성을 사용하여 flex-item의 주 바꿈의 여부를 결정할 수 있다. nowrap(기본값)을 사용하면 한 줄 내로 모든 item을 보여주려해서 width를 설정한 flex-item의 width를 강제로 줄이게 된다. 만약 해당 부분이 문제가 된다면 wrap을 사용하여 줄바꿈 할 수 있께 설정해야한다.

  • justify-content : flex-start, flex-end, center를 사용하여 flex-item을 어디서부터 시작할지 정할 수 있다. start의 경우 시작점, 즉 왼쪽부터 정렬하고 end는 오른쪽 정렬, center는 가운데에 정렬한다

  • align-content : stretch, flex-start, flex-end, center를 사용하여 flex-item를 여러 줄에 대한 수직 정렬에 관한 속성이다.
    align-content를 쓰지 않으면 container의 높이에 맞춰 여러 줄의 공간을 동일하게 분배하는데, align-content를 사용하면 item 높이에 맞춰 바로 밑에 위치할 수 있게 작동한다.

  • align-items : align-content가 여러 줄이라면 align-items는 한 줄에 대한 정렬 방법이다. content와 동일한 기능을 사용 가능하다.

Flex items에 관한 속성

  • order : 숫자를 사용할 수 있으며 flex item들의 순서를 정할 수 있다.
    기본적으로 ABCDE의 순서라면 각 항목에 order를 -3, 2, -5, 14, 7 이렇게 부여하면 순서가 CABED 이렇게 변경된다. 즉, 숫자가 작을 수록 앞쪽에 위치된다.

  • flex-grow : 숫자를 사용할 수 있으며 flex-item의 증가되는 width 비율을 정할 수 있다.

  • flex-shrink : 숫자를 사용할 수 있으며 flex-item의 감소되는 width 비율을 정할 수 있다.

  • flex-basis : auto와 각 단위로 지정할 수 있으며 flex-item의 공간 배분 전 기본 너비를 지정할 수 있다.

전환 효과

  • transition-property : all, 속성 이름을 사용하며 width, height 등 도 사용할 수 있다.
    transition : width 1s고 hover하면 width와 bgcolor가 바뀐다면 width만 1초동안 바뀌고 bgcolor는 즉시 바뀌게 된다.
  • transition-duration : 1s, .5s, 10s… 등을 사용 가능하며 전환이 되기까지의 시간을 사용할 수 있다.
    transition : width .5s, background-color 2s 처럼 여러가지 속성을 동시에 쓸 수 있다.
  • transition-timing-function : ease(느리게→빠르게→느리게), linear(일정하게), ease-in(느리게→빠르게), ease-out(빠르게→느리게), ease-in-out(느리게→빠르게→느리게) 등을 사용할 수 있으며 전환 효과의 타이밍을 조정할 수 있다.
    trasition: all .5s ease-in 이렇게 사용할 수 있다.

원근법 사용

transform : perspective를 사용하며 해당 요소에 원근범 함수를 사용할 수 있다. perspective의 경우 모든 transform 함수에서 가장 앞에 있어야만 한다.

다만 perspective는 자체적인 속성과 transform 내에 있는 속성 두 가지가 있다. perspective의 경우 관찰 대상의 부모 요소에 적용하고, transform : persperctove의 경우 관찰 대상 자체에게 원근법을 부여한다.

SCSS

  • SCSS란 CSS를 좀 더 활용하기 위한 도구이다.
  • 기존 SASS에서 조금 더 보완을하여 나온 것이 SCSS

  • .scss로 작업을 하면 npm 파일에서 자동으로 css 문법으로 컴파일하여 적용된다.
  • 기존 길어지던 선택자가 변수로 통합하거나 배경색, 폰트 사이즈 등을 변수로 지정할 수 있어 작업에 혼동이 덜하게 된다.

SCSS 기능

  • .scss 파일 내엔 css 요소를 변수로 설정할 수 있다. 또한 선택자 안에서 선언하면 해당 선택자 안에서만 사용 가능하다
  • 선택자를 중첩하여 사용할 수 있다.
  • 상위 선택자 참조 (&)를 통하여 hover, active, last-child등을 사용할 수 있다.
  • 동일한 이름의 속성은 중첩되는 이름 부분(name space)을 같이 쓸 수 있다.
  • +, -, *, % 등 사칙연산을 할 수 있다. 다만 /의 나누기는 사용할 수 없다. 대신 해당 연산을 괄호로 묶거나 변수에 값을 할당 후 엔 사용 가능하다.
    기존 사칙연산처럼 곱하기, 나누기, ( )부터 진행된다.

Mixin

재활용할 수 있는 타입의 구조이다. @include를 통해 불러올 수 있으며, 불러오면 설정한 속성을 붙여넣을 수 있다. 동시에 함수처럼 인자를 받아서 넣을 수 있으며 기본 값도 줄 수 있다.

반복문 @for

@for을 통하여 반복문을 사용할 수 있다. js가 아니기 때문에 숫자는 1부터 시작된다. 다만 반복되는 변수 i는 #{$i} 라고 보간하여 적어야 인자로 쓸 수 있다.

함수

@mixin, @function 을 통하여 함수를 사용할 수 있다. @mixin의 경우 css 코드의 모음이고, @function의 경우 여러 연산에 관련되었을 때 사용한다.

색상 내장 함수

  • mix(a,b) : a색상과 b색상을 섞어서 색이 나온다
  • lighten(a, num%) : a색상을 num%만큼 밝게 해준다
  • darken(a, num%) : a색상을 num%만큼 어둡게 해준다.
  • saturate(a, num%) : a색상을 num%만큼 채도를 밝게 한다.
  • desaturate(a, num%) : a색상을 num%만큼 채도를 어둡게 한다.
  • grayscale(a) : a색상을 회색으로 만든다.
  • invert(a) : a색상을 반전된 색상으로 만든다.
  • rgba(a, .num) : a색상을 .num만큼 투명하게 한다.

가져오기

@import를 통해 .scss 파일에서 다른 .scss 파일을 가져와 사용할 수 있으며 해당 함수나 mixin도 사용 가능하다. 확장자를 쓰지 않고 파일명만 적어도 되며 ,를 통해 여러 파일을 가져올 수 있다.

데이터 종류

  • String (bold, relative, “https://url.com”…)
  • Number (1, 2px, 100%, 5em…)
  • Color (red, #ff000f, rgba(0,0,0,.4)
  • Boolean
  • Null
  • List (orange, royalblue, yell …)
  • Map (o: orange, r: royalblue, y: yello…)

BFC (Block Formatted Context)

Block 요소의 영역을 잡는 계산. 어떠한 Block 요소의 영역을 잡아주는 계산이며 float:left 상황에서 부모 요소에 overflow를 사용하면 기존 float 상태인 Block에 부모 요소를 한 번 더 영역을 재계산해주는 것이다.

float의 부모 요소에 넣을 수 있는 BFC

  • overflow : hidden - overflow는 넘치는 영역에 대하여 값을 계산해주는 것이기 때문에 재계산이 가능하다
  • display : inline-block - inline-block의 경우 자식의 모든 요소를 inline-block으로 감싸서 잡아준다
  • displat : table-cell - table-cell도 위와 동일한 이유로 가능하다
  • position:absolute - position 은 float 층 중 가장 최상위 층에 위치해있어 영역이 다시 계산된다

회고

오늘도 이전과 같이 HTML, CSS 강의를 수강하였습니다. 다만 HTML 부분은 이제 끝나 주로 CSS에 관련된 부분이였고 float나 position 부분에 대하여 수업을 진행하였습니다.

대부분의 정의는 알고 있었으나 흥미로웠던 점이 BFC란 정의에 대해서입니다. 기존 float를 쓰면 요소가 붕 뜬 상태로 존재한다는 개념은 알고 있었으나 어떠한 원인과 과정을 통하여 작동하는지는 몰랐는데 확실하게 알아갈 수 있어서 좋은 내용을 들었다고 생각합니다.

제목 태그

제목 태그(<h1>, <h2>…)은 해당 페이지 의미의 구조를 표현한다. 주로 유명한 페이지는 본인들의 로고나 제목에, 유명하지 않은 페이지들은 본인들의 어필요소에 넣는게 좋다.

  • 네이버의 로고, 쇼핑몰의 상품명 등

Semantic

웹 표준에 기반한 뜻을 가진 태그들을 뜻한다.

  • <nav>, <section>, <article>, <aside>…

HTML Outline

HTML의 태그들 중 웹표준 상 깊이가 있는 태그들이 있다.

HTML5 Outliner 확장 프로그램으로 확인 시 무분별한 section이나 h1의 남발로 이 깊이가 다르게 되는 경우가 있는데 아래 예시와 같다

책의 목차와 같이 웹표준에 맞게 태그를 사용하는 것이 좋은 방법이다.

HTML, HTTP, Browser

어셈블리 이후 나타난 것이 **HTML, HTTP, Browser** 이다.

  • HTML = 쉽게 코드를 작성할 수 있고
  • HTTP = 쉽게 요청을 보내거나 받을 수 있고
  • Browser = HTTP를 통해 전달받은 정보를 사용자에게 보여줄 수 있다.

Browser 주소창에 넣는 문장이나 요청의 형식은 URL(Uniform Resource Locator)이라고 불리며 작성에 대한 약속된 기준이 있다.

URL의 작성 방식

  • scheme: 약속된 방식의 이름 (ftp, http …)
  • host: 접속하려는 페이지의 주소
  • port: 접속하거나 정보를 주고 받는데 쓰이는 통로의 구분
  • path: 해당 주소의 어떠한 파일로 통하는 경로
  • query: 추가로 요청할 때 무엇을 요청할지에 대한 정보

HTML

  • DOM을 구성하는 가장 기본적인 언어
  • 태그 안에 생성되어야하며, 태그와 태그 내에서 내용을 작성
  • 약속된 의미에 맞도록 태그를 사용해야 컴퓨터가 처리하기 좋음

DOCTYPE

마크업 언어에서 문서 형식을 정의하며, 웹 브라우저가 어떤 HTML 버전의 해석 방식으로 페이지를 이해하면 되는지 알려주는 용도

META

#Git Flow

## Git flow 작업 순서

- git flow init ⇒ git flow 초기화 셋팅
- git flow feature start {branch name} ⇒ 기능 branch를 생성 후 기능 개발 시작
- 개발이 끝나면 git flow feature finish {branch name}로 branch 제거
- git flow release start {release name} ⇒ 릴리즈 기능 시작
- git flow release finish {release name} ⇒ 릴리즈에 대한 확인 후 commit 메세지 작성, 해당 branch는 삭제되니 merge 메세지 작성
- 작업한 내용을 push를 안하면 github는 모르기때문에 git push -u origin {branch name} (여기서는 main branch가 아닌 작업했던 branch name을 기입)
- 이후 main branch로 가서 똑같이 git push origin main
- tag를 알리기 위해 git tag  를 입력 후 git push —tags로 버전도 같이 push
**ex) 첫 git flow tags link :** [https://github.com/dbstjddbwls/branch-practice/tags](https://github.com/dbstjddbwls/branch-practice/tags)

## Git Rename

- mv {file} {rename file}을 하고 git status를 하면 rename이 아니라 delete라고 인식한다.
- 이를 방지하고싶다면 git mv {file} {rename file}이라고 앞에 git을 넣어줘야함

## Git Restore

- 수정 후 commit까지 했는데 다시 되돌려야할때
- git restore {file name} 를 통해 바로 이전버전 커밋으로 되돌림

## Git Unstaging

- add를 취소하기
- add한 파일을 다시 unstaging하려면 git reset HEAD {file name}
    - 수정한 내용이 사라진건 아님

- restore와 reset의 차이점은 내용을 이전 버전으로 수정 하냐안하냐의 차이로 나뉘어진다
- 단 reset은 내 repo에서 삭제해도 다른 팀원의 repo에는 살아있을 수 있기 때문에 쓰지 않는걸 권장
    - reset의 대안은 **revert**를 사용

## Git Revert

- git revert —no-commit HEAD~{num}
- {num} 수 만큼 가장 최근 기준으로 이전에 커밋한 내용을 되돌림
- reset과의 다른 점은 revert의 경우 가장 최근의 커밋 내역까지 되돌리며 삭제까지 진행.
- 즉 파일 생성→ add→ status→ commit  순으로 보통 진행했다면 commit →status →add →파일 삭제로 되돌아감
- revert했다면 꼭 사유를 commit 메세지에 적을 것
