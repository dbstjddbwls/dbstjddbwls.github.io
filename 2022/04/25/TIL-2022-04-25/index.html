수업 내용

Global Context

코드 구현을 위한 글로벌 환경을 설정하는 것. 정해진 내부 순서를 마친 뒤 다시 Global Context를 실행하여 다른 환경을 세팅한다.

Global Context 순서

  • 기본 window 관련 함수, 사용자 설정 함수 선언
  • var의 참조자 선언
  • Outer 환경 참조 및 this 객체 선언
  • Object 객체 선언, let이나 const에 변수에 값을 할당

let과 var의 차이점 (블록 스코프)

조건문 안에서 재선언 후 로컬이 아닌 전역에서 호출하였을 때 값이 달라진다.

let과 const의 경우 한 블록(동일한 중괄호) 안에 같은 이름으로 let 변수명을 또 한 번 선언하면 오류가 난다. 그러나 var의 경우엔 이러한 오류가 나지 않는다.

즉 전역 스코프냐 블록 스코프냐의 차이인데, let과 const는 블록이 다르면 새로운 화이트보드 위에 새로 그리게 되는 것이고, var는 그것이 조건문이나 반복문 밖에도 영향을 끼치게 되는 것이다.

단순히 var는 재선언, let이나 const의 경우 같은 블록 스코프에선 재선언이 불가하며 다른 블록일 경우 재선언보단 동일한 이름의 새로운 변수가 탄생하는 것이다.

이 경우 에러가 나오는데 이유는 초기화가 되지 않아서이다. var였으면 실행 전 지역 변수에 없으니 전역 변수에서 값을 가져왔을 것이지만, let이나 const의 경우 지역 내에서 전역 let이나 const의 값을 가져오고 싶다면 초기화를 우선적으로 해야한다.

다만 선언을 어떠한 형식으로 하냐에 따라 지역 변수가 없다면 전역 변수에서 받아오는 것이 가능하다.

요약) let과 const는 초기화가 되지 않는다. function name()으로 선언을 하면 null로 초기화가 된다.

함수 스코프

블록 스코프와 같이 function에서 받은 인자값은 외부에서 해당 인자값의 변수명으로 불러올 수 없다. 같은 인자의 이름을 사용하더라도 지역인자로만 사용되어 전역의 값으론 사용이 불가능하여 꼬이지가 않는 것이다.

호이스팅

위와 같이 const로 선언을 하였다면 const는 Global Context의 순서에 의해 const 선언이 함수 실행보다 뒤에 있어서 오류가 난다.

그러나 function의 선언의 경우 Global Context 순서 중 가장 첫 사이클, 준비 단계에 초기화 후 선언되기 때문에 (선언형 방식) 가능해진다.

호출이 선언보다 위에 있어도 Global Context의 순서에 따라 끌어오는 것을 호이스팅이라고 한다.

함수 코드 평가

함수의 단위, 즉 하나의 함수가 생길 때마다 함수 실행 컨텍스트가 생성된다. 가장 첫 단계인 콜스택에 하나씩 쌓이는 것이다.

만약 콜스택 1의 함수에서 a를 console.log를 하라 하였을 때 해당 함수 내에 지역 변수로 없다면 다른 함수나 전역으로 가서 찾게된다. 이후 함수가 실행되면 콜스택과의 관계를 끊어버리며 스택의 층이 변하게 된다. 이 때문에 함수 안에서 함수 밖에는 접근이 가능하나, 함수 밖에서 함수 안의 지역 변수에 접근을 할 수가 없다.

회고
