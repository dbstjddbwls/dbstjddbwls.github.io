수업 내용

Cookie

  • 브라우저(Document)에서 저장되는 데이터의 일종
  • 도메인 단위로 저장된다.
  • 용량과 갯수에 제한이 있으며 만료일 설정이 가능하다.
  • 자동으로 HTTP Request에 포함된다.

Session

  • 쿠키에는 서버가 발급해준 id만 저장하고, 데이터는 서버에 저장한다.
  • Session 장점
    • 서버측에서 데이터의 만료기간을 컨트롤 가능
    • 쿠키의 용량이나 갯수 제한을 해결
    • 데이터 자체를 서버에 저장함으로 보안에 유리
  • Session 단점
    • Session id를 서버에 저장해야하므로 메모리 차지
    • Server가 데이터를 들고있어, 인증이 필요없는 데이터에 접근할 때도 서버 상태나 네트워크에 영향을 받음

Web Storage

  • LocalStorage : 직접 삭제하지 않으면 탭이나 브라우저 종료시에도 데이터가 유지된다.
  • SessionStorage : 탭 혹은 브라우저 종료시엔 데이터가 사라진다.
  • Cookie의 단점 해결을 위해 도입이 되었다.

Redux-Persist

  • Redux Store의 데이터와 Storage를 연결해주는 라이브러리
  • Cookie, LocalStorage, Session Storage 등 다양하게 지원
  • Store 중 일부 Reducer를 선택해 저장할 수 있으며, Property 단위로 선택할 수 있도록 Whitelist, Blacklist도 지원한다.
  • 사용하기 위해선 npm i redux-persist를 해야한다.
  • store 파일과 해당 Storage 값을 전달받을 태그를 index 파일에서 컨트롤 해줘야 한다.

회고

오늘은 Storage에 대해 중점으로 공부한 것 같습니다. Cookie와 Session, Storage 이 3가지를 어떻게 사용하고 언제 사용하는지 구분이 가장 중요한 것 같습니다.

개인적으로 쿠키의 경우 기간 제한을 두고 유저가 여차하면 변경할 수 있는 팝업 창 또는 공지 배너 등에 사용을 하고 세션의 경우 유저 데이터를 서버에서 저장하고 가져오는 것이기 때문에 보안이 필요한 로그인이나 회원 정보, storage는 기간에 상관없이 계속 저장해두는 것이 필요한 자동 로그인 또는 브라우저 종료 후 재접속시에도 유지되야하는 기능 등에 쓸 수 있을 것 같습니다.
