TypeScript Compiler

  • Compilation Context : 어떤 파일을 컴파일 하거나 컴파일 하지 않을 것인지 구분하고, 변환 시 어떠한 옵션을 사용하는지 정함
  • CompileOnSave : true / false(defual)로 구분하며, true로 설정하였을 때 컴파일을 어느 프로그램에서 진행하느냐를 정함
  • extends : 주로 상속받을 때 사용된다. 파일 경로명이 String Type으로 들어가야하며, ts 2.1 버전 이상부터 사용 가능한 속성이다. 특수한 설정 파일을 만들어서 tsconfig 파일을 수정없이 사용할 수 있다.
  • files, include, exclude : 이 3가지 속성 다 설정이 없으면 실행 폴더 내 모든 파일을 컴파일 한다. 어떤 파일이나 어떤 디렉토리를 컴파일 하는지 정하는 속성이다.
    • files
      • 상대 혹은 절대 경로의 리스트 배열이다
      • exclude보다 강한 영향력을 행사한다.
    • include, exclude
      • glob 패텁을 사용한다. (.gitignore 과 유사함)
      • include는 exclude보다 약하지만 *를 사용하면 .ts/ .tsx/ .td.ts 만 include 한다.
      • exclude는 설정을 안하면 4가지(node_modiles, bower_components, jspm_package,)를 default 값으로 제외하고 컴파일한다. 단 은 include에 포함되어도 제외한다.
  • CompileOptions

    • @types

      • TypeScript 2.0부터 사용 가능해진 내장 시스템이다.
      • node_modules/@types 라는 모든 경로를 찾아서 사용한다.
      • 배열 안의 모듈이나 node_modules/@types 안의 모듈 이름에서 찾아온다.
      • [ ] 같은 빈 배열을 넣으면 false 처리가 된다.

    • typeRoots

      • 해당 배열 안에 들어있는 경로들 아래에서만 가져온다.
      • types와 typeRoots는 같이 사용되지는 않는다.

    • target

      • 어떠한 runTime에서 실행하는지 결정해준다.
      • defualt 속성의 값을 기준으로 어느 버전으로 컴파일 할 지 결정한다. 아무것도 지정하지 않으면 es3 기준으로 컴파일을 실행한다.

    • lib

      • 실행하려는 환경에 맞게 기본 속성들을 설정한다.
      • lib를 지정하지 않았을 때 target이 es3이면 lib.d.ts를, es5면 dom, es5, scripthost를, es6면 dom, es6, dom.iterable, scripthost를 디폴트로 사용한다.
      • lib를 지정하면 해당 lib 배열로만 라이브러리를 사용한다.

    • outDir

      • 컴파일 내용을 출력할 디렉토리

    • outFile

      • 단일파일로 출력 시 파일명

    • rootDir

      • 입력파일의 rootDir 설정

    • strict

      • 엄격한 타입 검사를 실행하는 옵션

      • —nolmplicitAny

        • 타입스크립트가 추론에 실패한 경우, any가 맞으면 any라고 지정하라함.
        • 아무것도 쓰지 않으면 에러
        • 해당 오류를 해결하면, any라고 지정되어 있지 않은 경우는 any가 아닌 것임

      • suppressImplicitAnyIndexErrors

        • noImplicitAny를 사용할 때 인덱스 객체에 인덱스 signature가 없는 경우 오류가 발생하는데 이를 예외 처리

      • —noImolicitThis

        • 명시적이지 않게 any 타입을 사용하여, this 표현식에 사용하면 에러가 발생함
        • 해결을 위해선 매개변수 인자 첫 자리에 this를 놓고, this에 대한 타입을 선언해야한다. class를 사용하여도 가능하다.
        • js에서 인자에 this를 넣으면 예약어라 에러가 난다.
        • call, apply, bind 등과 같이 this를 대체하여 함수 콜을 하는 용도로 쓰인다.
        • this는 보통 any로 선언한다.

      • —strictNullChecks

        • null 및 undefined 값을 모든 타입이 가질 수 없고, 가지려면 union type이나 any는 null과 undefined를 가진다.
        • 예외적으로 void는 undefined를 가진다.
        • 해당 옵션을 적용하지 않고 계속 사용하다보면 정확히 어떤 타입이 오는지 헷갈릴 수 있다.
        • null과 undefined를 사용하는 경우 신중하게 사용해야하며 되도록 정확한 타입을 지정하려는 노력을 해야한다.

      • —strictPropertyInitialization

        • 정의되지 않은 클래스의 속성이 생성자에서 초기화되었는지 확인한다.
        • 해당 옵션을 사용하려면 —strictNullChecks를 사용해야한다.
        • constructor를 사용 안하는 경우는 다른 함수로 이니셜라이즈 하는 경우엔 사용하지 않는다. (async)

      • —alwaysStrict

        • 각 소스 파일에 대해 js의 strict mode로 코드를 분석하고 엄격하게 사용을 해제한다. 즉 타입의 선언을 안해도 되는 것.

Interface

  • 타입을 정리하는 하나의 데이터 묶음이다.
  • 변수나 인자에 타입 선언 대신하여 넣을 수 있다.
  • 해당 키워드는 컴파일 타임에만 사용되며 컴파일이 되면 js 파일에는 보여지지 않는다.

Interface 작성 시 Option

  • ? : 해당 인자가 값이 들어올 수도 있고 안들어올 수도 있다.
  • [index: string] : 해당 인자 내에 어떠한 속성 값도 넣을 수 있다. 즉 interface내에 설정한 이름이 아니여도 무작위로 넣을 수 있게된다.

Classes

  • object를 만드는 설계도
  • 클래스 이전에 object를 만드는 기본적인 방법은 function
  • Javascript에서도 class 는 es6부터 가능
  • TypeScript에서는 class도 사용자가 만드는 타입의 하나이다.
  • this를 이용하여 만들어진 object를 가리킬 수 있다.
  • es5 이하 버전으로 컴파일하면 function으로 변경된다.

Constructor & initialize

  • 생성자 함수가 없으면 디폴트 생성자가 불린다.
  • 생성자 함수가 있으면 디폴트 생성자는 사라진다.
  • strict 모드에서는 프로퍼티를 선언하는 곳이나 생성자에서 값을 할당해야한다.
  • 프로퍼티를 선언하는 곳이나 생성자에서 값을 할당하지 않는 경우에는 !를 붙여 위험 표시를 한다.
  • 클래스의 프로퍼티가 정의되어있어도, 값을 넣지 않으면 undefined다.
  • 생성자 자체에는 async를 설정할 수 없다.

Acces Modifiers

  • 외부에서 접근할 수 있는지, 상속받은 요소에서 접근할 수 있는지, 내부에서만 접근 가능한지 설정할 수 있다.
  • 클래스 내부의 모든 곳에 설정이 가능하다.
  • public : 가장 기본 설정이며, 설정하지 않으면 public이 된다. 외부에서도 접근을 가능하게 한다.
  • private : 클래스 외부에서, 심지어 호출하는 부분에서도 인자를 주는 것마저도 접근할 수 없다. javascript에서는 지원하지 않아 예전부터 프로퍼티나 메서드 앞에 _를 붙여서 사용하였다.
  • protected : 상속받은 요소에서만 접근이 가능하다.
