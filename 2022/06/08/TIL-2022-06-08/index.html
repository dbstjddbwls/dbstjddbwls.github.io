수업 내용

Type System

  • 타입을 명시적으로 지정 가능하다.
  • 타입을 명시적으로 지정하지 않으면, 타입스크립트 컴파일러가 자동으로 any라고 추론한다.
  • 타입이란 해당 변수가 할 수 있는 일을 결정해주는 것이다.

Sub Type

  • 작은 타입에는 큰 타입을 넣지 못한다.
  • 하지만 반대로 큰 타입에는 작은 타입을 대입시킬 수 있다.
  • 단 any의 경우 에러가 발생하지 않는다.
  • 같은 타입이거나 서브타입 또는 공변( | )인 경우 할당이 가능하다.
  • 함수의 매개변수 타입만 같거나 슈퍼타입인 경우 할당이 가능하다.

TypeScript Compilation Context

타입스크립트 컴파일러가 어떤 기준을 가지고 JS로 번역할 것인지 결정해주는 하나의 맥락이자 기준이다.

Interface Indexable Type

  • 어떠한 값이 들어와도 괜찮은 타입
  • [ index : string ] : any 처럼 사용한다.
  • 이후 어떤 key 값과 value로 오던간에 index 안에 넣어버린다.

Class in interface

  • class를 선언할 때 className 뒤에 implements InterfaceName을 적으면 최상단에 type을 지정해줄 수 있다.
  • type 지정 후 그 밑은 일반 class 처럼 사용이 가능하다.
  • 단 implements class 내에서 선언한 type을 쓰기 위해선 constructor 내에서 선언해줘야 한다.
  • 이렇게 만든 class는 기존 interface를 참조하여 쓰는 것이다.

extends TypeScript

  • 일종의 확장 기능으로 기존 interface를 참조하여 추가로 타입을 더 붙힐 수 있다.
  • 참조하여 만든 interface를 이용하여 변수를 만들 수 있으며, 동일한 값이 반복되는 interface가 있을 때 사용한다.

Middleware

  • 프로그램(JS Code)과 프로그램 사이에 들어가는 또 다른 프로그램
  • 특징 : Chain 구성(연속적인 사용) 가능
  • Redux에선 store와 action을 파라미터로 받는 함수 action이 dispatch 되면 등록된 middleware에 순차적으로 입력된다.
    모든 middleware을 통과하면 reducer에 도착한다.
  • store를 파라미터로 받았으니 원한다면 getState나 dispatch도 가능하다.
  • 미들웨어는 모든 reducer들로 action이 전달될 때 거쳐가는 기능이며, 특정 reducer에만 적용하고 싶다면 reducer 내에 함수를 적는게 좋다.
  • 필수적으로 next(action)을 해줘야 원래 Reducer로 정보를 전달해준다.

Middleware 사용하기

  • Arrow Function을 사용한 경우
  • 함수 선언식을 사용한 경우
  • 미들웨어 등록하기
  • 예제
    • 원래 reducer인 userReducer가 받아야할 값을 미리 받아 쓸 수 있다.
    • next(action)을 쓰지 않으면 전달되지 않는다.

Redux-thunk

  • Redux에 함수를 Dispatch(= action을 전달)하면 Reducer로 전달하지 않고(= next를 호출하지 않고) 해당 함수를 실행시켜주는 간단한 Redux Middleware

  • 해당 전달 action이 함수면 바로 실행하고, 함수가 아니면 값을 전달해주는 기능이다.

  • 기존 dispatch({ … }) 처럼 객체만 넘겼다면 이젠 실행함수 형식 자체로 넘길 수 있다.

  • 되도록 view file이 아닌 store 파일에서 작업하는게 좋다.

  • thunk : dispatch와 getState를 받는 함수를 create 하는 함수

회고

진상현 강사님의 마지막 React 강의가 끝났습니다. 뒤로 갈수록 난이도가 기하급수적으로 상승하여 이번 수업의 경우 거의 10%도 이해하지 못한 것 같습니다. 마지막 수업이라 그러한지 너무 급하게 끝내시려는 모습도 보인 것 같습니다.

thunk의 경우 하나의 도구로써 자주 사용이 안될 수 있지만 현업에서 언젠가 막혔을 때 키가 될 수 있는 요소라고 생각합니다. 특히 로딩 중임을 띄워야 할땐 더욱이 그렇게 느끼는 바입니다. 그러한 기능을 실습에서 50명 중 45명 이상이 막혀하는 실습인데 풀이해석과 설명 없이 그저 끝내고 끝인게 너무 아쉬웠습니다.
