수업 내용

TypeScript Classes

  • Object를 만드는 하나의 설계도
  • 클래스 이전에 Object를 만드는 방법은 function이였다.
  • TypeScript에서는 클래스도 사용자가 만드는 하나의 타입이다.

Start Class

  • Class의 첫 글자는 무조건 대문자로 적는다. (구분을 위한 규칙)

  • 사용할 땐 new ClassName()을 변수에 할당하여 사용한다.

  • npx tsc —init 명령어를 통해 config 파일을 생성할 수 있다.

  • .ts파일에서 작업을 완료한 후 npx tsc 명령어를 입력 하면 자동 컴파일링 되어 js 파일로 변환해준다.
    단 tsx의 경우 리액트 실행 시 자동 변환이 된다.

  • Class 내부에 constructor(param){ } 를 통하여 전달받는 인자에 대하여 생성자를 만들어 인자 자체를 Class에 귀속시킬 수 있다.

Constructor & Initialize

  • Class 내에서 어떠한 값을 선언하기 위해선 타입을 꼭 선언해줘야 한다.
  • 타입을 선언할 때 값(key) 부분에 !나 ?를 붙혀 값의 필수성을 조정할 수 있다.
    • !가 붙으면 이 값은 무조건 할당이 되어있으니 에러가 뜨지 않게 하라는 기능이다.
    • ?가 붙으면 해당 값이 인자로 올 수도 있고 없을 수도 있다고 알려준다.
  • 생성자에는 async를 설정할 수 없다.

Getter & Setter

  • 클래스 내 함수의 선언 앞에 get을 붙히면 return하는 값이 필수로 있어야하며, return 전에 어떠한 코드를 넣을 수 있다.
  • 클래스 내 함수를 호출할 때 set을 붙힌다면 해당 setter 함수가 호출이 된다.
  • get과 set을 이용한다면 아래 코드에서 _name 변수 생성자에 private를 써도 접근이 가능해진다. (get과 set은 Class A 안에서 작동하기 때문에 직접 _name에 접근하는게 아니기 때문)
  • 단 get을 없애면 두 console.log 다 undefined가 뜨고 set을 없애면 재할당이 불가능해진다.

Readonly

  • Getter와 비슷한 기능이지만 읽기 전용으로, 즉 불러오는 기능만 지원하며 수정이 아예 불가하게 만들 수 있다.
  • 변수 선언 시 public이나 private 과 변수명 사이에 readonly를 붙히면 된다.
    예시) public readonly name: …

Indexable

변수 선언시 대괄호로 감싸서 선언을 하면 해당 Class에 추가로 값을 선언하면 대체되어 들어간다.

Static

  • public이나 private과 변수명 사이에 static을 넣으면 해당 Class에서 요소를 가져와 사용할 때 굳이 변수에 new ClassName으로 할당하고 사용하지 않고 바로 ClassName.staticProps 처럼 사용할 수 있다.

  • 변수든 함수든 사용없이 사용이 가능하다.

  • 해당 Class 안에서도 사용이 가능하다.

Generics

  • 소괄호 안에 Generic Name을 넣어 사용하는 타입 선언 방법

  • 자동으로 해당 타입을 추론해서 보여준다.

  • 배열로 받고 싶다면 인자 뒤에 배열 표시를 해주면 된다.

  • 다른 타입을 2개 동시에 사용하는 방식은 Tuple이라고 한다.

  • Function 형식으로 Generic을 선언할 수 있다. 단 이렇게 선엉한 Generic은 실제로 사용할 때 구현체에 동일하게 Generic 선언을 한 번 더 해줘야한다.

  • function에서 사용이 가능한 만큼 Class에서도 사용이 가능하다.

  • keyof를 통하여 유니온 타입이면서 동시에 조건을 만족해야할 때 문제를 해결할 수 있다.

회고

타입스크립트의 복습 수업이 끝났습니다.

타입스크립트를 지금까지 배워오면서 드는 생각은 이렇게 안전해도 되나와 이렇게 복잡해도 되나의 반복이였던 것 같습니다.

항상 코드를 짜며 이 코드가 맞는지, 어디서 이런 오류가 나는지 몰랐던 적이 많은데 타입스크립트가 그 오류를 코드 작성 과정에서 보여주는 점이 너무 매력적입니다.

이번 파이널 프로젝트에도 1순위 희망 기업이 타입스크립트로 코드 작성을 원하는 기업이였고 그러한 만큼 최대한 지금보다 더 익숙해져야 실무에서 적어도 문법오류 없이 쓸 수 있을 것 같습니다.

기본적인 문법은 다 배웠다고 생각합니다. 앞으로 남은 길은 계속 쓰며 익숙해지고, 문제가 생겼을 때의 해결능력을 늘리는 것만이 남은 것 같습니다.
